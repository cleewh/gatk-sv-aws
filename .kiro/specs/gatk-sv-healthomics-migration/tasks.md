# Implementation Plan: GATK-SV HealthOmics Migration

## Overview

Convert the GATK-SV HealthOmics migration design into a series of prompts for a code-generation LLM that will implement each step with incremental progress. Each task builds on the previous ones and ends with wiring things together, so no hanging or orphaned code is left un-integrated. Tasks focus only on writing, modifying, or testing code and concrete artifacts.

Implementation language is **Python 3.11+**, consistent with the design's `def`-style API sketches, the existing `kiro-life-sciences/` Python package, and the already-provisioned Hypothesis test infrastructure (`.hypothesis/`). All property-based tests use Hypothesis with ≥100 iterations. Integration work uses the AWS HealthOmics MCP tools listed in design §Testing Strategy → Integration Tests and §Deployment & Operational Procedures.

All tasks below are required; nothing is optional. Each task cites the requirement IDs and design sections it advances.
